This fork of CCCC adds several some new metrics to the original CCCC project, as well as enhancing the HTML output to make it more suited for direct integration into external build environments.

The new metrics "Lines of Code per Method" and "Maximum Lines of Code per Method" highlight excessively long methods which should be divided up. Note that large numbers of simple accessor methods may unfairly weigh the lines count towards zero.

The HTML rendering now includes some support Javascript to

  • provide descriptive tooltips for each column header, for the forgetful (including the author)
  • keep a copy of the table head at the top of the page when scrolling through a long table
  • sort table in response to column header clicks

The intent is that the HTML output should be suitable for fast integration with a build environment such as Jenkins.

Example output from CCCC4 run against itself.
